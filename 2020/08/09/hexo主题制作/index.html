Hexo基础

在我们写我们的第一个模版文件前，先看一下Hexo博客生成的基础。

界面类型

我们可以在我们的主题里定义6种类型的界面，每个在公共文件夹生成的HTML界面都可以对应到这6种类型的界面。

index 无 这是博客的主页，主接入点，在我们的例子里将会用于展示文章摘录的列表
post index 这是博文的细节页，展示完整的博文，带有评论功能
page index 这是
archive index 这是归档页，这会用来展示你博客里的全部博文，但是只是标题和链接
category archive 这是分类页，和归档页差不多，但是经过分类来筛选
tag archive 这是标签页，和归档页差不堵，但是通过标签来筛选

在这部分的教程中，我们将建一个index页。
在生成index页的时候，Hexo会查看文件名为index.ejs,post.ejs,page.ejs等等，这些模版被用于渲染生成静态的HTML页面。

通用布局

Hexo支持使用common layout文件，这会被全部模版所使用。

这些被命名为layout.ejs的文件，就像内容的外包装，被上面不同的的界面类型模版所渲染。
在我们的主题里，layout会包含<html>和<head>标签，header和menu，还有footer和sidebar，基本所有的元素在全部的界面都是通用的。

不同的界面模版只会用于创造实际的内容，这主要被放在我们的main container里

变量

在我们的模版内，我们有方法接入某些变量，这些可视化是Hexo引擎自带的，这是一些例子：

  • Site
    site包括很多信息，比如说，用site.posts，我们可以进入博客里的全部博文，如果我们想要在小窗口展示统计数据，这将会非常有用。
  • Page
    page是主变量，而且包含很多和最近页相关的信息，包括博文的标题，日期，内容等等。
  • Config
    config是一个JavaScript对象，代表博客的主_config.yml文件
  • Theme
    theme是一个JavaScript对象，代表主题的_config.yml文件
