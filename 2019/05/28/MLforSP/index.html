Introducation

语音的特点: 大部分语音信号虽然是short‐term stationary，但是随时间频繁变化。

我们应该怎么用什么模型去描述这些语音的序列数据呢？

首先想到的是Joint Probability这个模型，我们可以通过joint probabilities来描述序列数据之间的独立性。

但是上面的公式的复杂度，会随着时间的增加而增加。所以我们需要对此进行简化，所以我们想到了两个新的模型。

  1. 通过Hidden Markov Model引入discrete latent variable。
  2. 通过Linear Dynamical System引入continuous latent variable。

ML Training for HMM Parameters

原因： 因为连续的数据点是相关的，我们不可能独立的对z进行求和。而且，上面的总和对应着网格图指数级条路径的求和，复杂度太高了。我们可以通过利用基于HMM的数据点“条件独立性”，来降低复杂度。

使用EM算法进行优化，Baum‐Welch Algorithm。

EM算法用于优化HMMs的大概流程

  1. 初始化参数集$\boldsymbol{\theta}_ {0}\rightarrow \boldsymbol{\theta} ^ { old }$。
  2. E Step: 找到Posterior Probability $p \left( \boldsymbol{Z} | \boldsymbol{X} ; \boldsymbol{\theta} ^ { \text { old } } \right)
    $
  3. M Step:求整个数据集的log likelihood的期望


找出新的参数集，使得数据集log-likelihood的期望最大。


然后将老参数集用新参数集代替，检查是否收敛，如果不收敛的话就返回E Step继续迭代，如果收敛的话，就结束。
